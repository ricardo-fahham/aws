# S3 Commands

Os comandos da **AWS CLI** para o serviço **Amazon S3** permitem criar e gerenciar buckets, além de enviar, baixar, sincronizar e remover arquivos. Abaixo está a descrição de cada comando com exemplos práticos.

## 1. `aws s3 ls`

**Descrição:**

Lista todos os buckets S3 pertencentes à conta AWS.

**Exemplo:**

```bash
aws s3 ls
```

**Exemplo de saída:**

```text
2025-01-10 15:30:10 empresa-backup
2025-03-22 09:18:44 projeto-web
2025-05-14 12:05:17 logs-aplicacao
```

---

## 2. `aws s3 ls s3://bucket-name`

**Descrição:**

Lista os arquivos e pastas armazenados em um bucket específico.

**Exemplo:**

```bash
aws s3 ls s3://empresa-backup
```

**Saída:**

```text
2025-06-10 08:30:12      15234 arquivo1.txt
2025-06-10 08:32:45     984512 backup.zip
                           documentos/
```

Também é possível listar o conteúdo de uma pasta:

```bash
aws s3 ls s3://empresa-backup/documentos/
```

---

## 3. `aws s3 mb s3://bucket-name`

**Descrição:**

Cria um novo bucket S3.

**Exemplo:**

```bash
aws s3 mb s3://meu-bucket-exemplo
```

Criando em uma região específica:

```bash
aws s3 mb s3://meu-bucket-exemplo \
    --region us-east-1
```

**Resultado:**

```text
make_bucket: meu-bucket-exemplo
```

---

## 4. `aws s3 cp file.txt s3://bucket/`

**Descrição:**

Envia um arquivo local para um bucket S3.

**Exemplo:**

```bash
aws s3 cp arquivo.txt s3://empresa-backup/
```

Enviando para uma pasta:

```bash
aws s3 cp arquivo.txt s3://empresa-backup/documentos/
```

**Resultado:**

```text
upload: ./arquivo.txt to s3://empresa-backup/arquivo.txt
```

---

## 5. `aws s3 cp s3://bucket/file.txt ./`

**Descrição:**

Baixa um arquivo do S3 para o computador local.

**Exemplo:**

```bash
aws s3 cp s3://empresa-backup/arquivo.txt ./
```

Baixando para uma pasta específica:

```bash
aws s3 cp s3://empresa-backup/arquivo.txt Downloads/
```

**Resultado:**

```text
download: s3://empresa-backup/arquivo.txt to ./arquivo.txt
```

---

## 6. `aws s3 rm s3://bucket/file.txt`

**Descrição:**

Remove um arquivo do bucket.

**Exemplo:**

```bash
aws s3 rm s3://empresa-backup/arquivo.txt
```

**Resultado:**

```text
delete: s3://empresa-backup/arquivo.txt
```

Para remover recursivamente uma pasta:

```bash
aws s3 rm s3://empresa-backup/documentos/ --recursive
```

---

## 7. `aws s3 sync . s3://bucket/`

**Descrição:**

Sincroniza os arquivos de um diretório local com um bucket S3.

Apenas arquivos novos ou modificados são enviados.

**Exemplo:**

```bash
aws s3 sync . s3://empresa-backup/
```

Sincronizando uma pasta específica:

```bash
aws s3 sync ./site s3://meu-site/
```

Excluir do bucket arquivos que não existem mais localmente:

```bash
aws s3 sync ./site s3://meu-site/ --delete
```

---

## 8. `aws s3 sync s3://bucket/ ./`

**Descrição:**

Sincroniza um bucket S3 com um diretório local.

Baixa apenas arquivos novos ou alterados.

**Exemplo:**

```bash
aws s3 sync s3://empresa-backup ./
```

Baixando para outra pasta:

```bash
aws s3 sync s3://empresa-backup Downloads/
```

---

## 9. `aws s3 rb s3://bucket-name`

**Descrição:**

Remove um bucket S3.

O bucket precisa estar vazio.

**Exemplo:**

```bash
aws s3 rb s3://meu-bucket-exemplo
```

**Resultado:**

```text
remove_bucket: meu-bucket-exemplo
```

Para remover um bucket e todo o seu conteúdo:

```bash
aws s3 rb s3://meu-bucket-exemplo --force
```

> **Atenção:** A opção `--force` exclui todos os objetos do bucket antes de removê-lo.

---

## 10. `aws s3 presign s3://bucket/key`

**Descrição:**

Gera uma URL pré-assinada (Pre-Signed URL) para acesso temporário a um objeto do S3, sem necessidade de autenticação adicional.

**Exemplo:**

```bash
aws s3 presign s3://empresa-backup/relatorio.pdf
```

**Resultado:**

```text
https://empresa-backup.s3.amazonaws.com/relatorio.pdf?X-Amz-Algorithm=...
```

Definindo o tempo de validade da URL (em segundos):

```bash
aws s3 presign s3://empresa-backup/relatorio.pdf \
    --expires-in 3600
```

Nesse exemplo, a URL será válida por **1 hora**.

---

# Resumo dos comandos

| Comando                             | Finalidade                                       |
| ----------------------------------- | ------------------------------------------------ |
| `aws s3 ls`                         | Lista todos os buckets da conta.                 |
| `aws s3 ls s3://bucket-name`        | Lista os objetos armazenados em um bucket.       |
| `aws s3 mb s3://bucket-name`        | Cria um novo bucket S3.                          |
| `aws s3 cp arquivo s3://bucket/`    | Envia um arquivo para o bucket.                  |
| `aws s3 cp s3://bucket/arquivo ./`  | Baixa um arquivo do bucket.                      |
| `aws s3 rm s3://bucket/arquivo`     | Remove um arquivo do bucket.                     |
| `aws s3 sync . s3://bucket/`        | Sincroniza arquivos locais com um bucket S3.     |
| `aws s3 sync s3://bucket/ ./`       | Sincroniza um bucket S3 com um diretório local.  |
| `aws s3 rb s3://bucket-name`        | Remove um bucket vazio.                          |
| `aws s3 presign s3://bucket/objeto` | Gera uma URL temporária para acesso a um objeto. |

### Fluxo típico de uso

Um fluxo comum de gerenciamento de arquivos no Amazon S3 utilizando a AWS CLI é:

1. Criar um bucket com `mb`.
2. Verificar os buckets disponíveis usando `ls`.
3. Enviar arquivos com `cp`.
4. Consultar o conteúdo do bucket com `ls s3://bucket-name`.
5. Sincronizar diretórios locais e o bucket com `sync`.
6. Baixar arquivos necessários usando `cp`.
7. Remover arquivos antigos com `rm`.
8. Gerar URLs temporárias para compartilhamento usando `presign`.
9. Excluir o bucket quando ele não for mais necessário com `rb` (ou `rb --force` para remover também seu conteúdo).
